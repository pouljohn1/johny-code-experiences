React apps are built from small, reusable building blocks called components. Each component describes a piece of UI and how it should render.

## Learning Objectives

* Define a React component and explain what JSX is at a high level
* Create your first component and render it to the page
* Compose components together to build richer UI

## What Is a Component

A React component is usually a JavaScript function that returns UI described with JSX.

```jsx
function Greeting() {
  return <p>Hello, React!</p>;
}
```

* Components must start with a capital letter: `Greeting`, not `greeting`.
* They return exactly one root element. Use a fragment `<>...</>` to group children.

## Rendering Your First Component

In a Vite app, `main.jsx` mounts your root component into the DOM.

```jsx
// main.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App.jsx';

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

Your `App.jsx` can render your new component:

```jsx
// App.jsx
function Greeting() {
  return <p>Hello, React!</p>;
}

export default function App() {
  return (
    <main>
      <h2>Welcome</h2>
      <Greeting />
    </main>
  );
}
```

## Anatomy of a Functional Component

* Input: props (data passed in)
* Body: plain JavaScript
* Output: JSX (what to render)

```jsx
function UserTag(props) {
  // props is an object, e.g., { name: 'Ada' }
  return <span className="tag">@{props.name}</span>;
}
```

You‚Äôll learn more about props and JSX in the next chapter: [JSX and Props](./02-jsx-and-props.mdx).

## Composition: Building UIs from Pieces

Compose components like LEGO to build complex UIs:

```jsx
function Card({ title, children }) {
  return (
    <section className="card">
      <h3>{title}</h3>
      <div className="content">{children}</div>
    </section>
  );
}

function PrimaryButton({ children }) {
  return <button className="btn btn-primary">{children}</button>;
}

export default function App() {
  return (
    <Card title="Getting Started">
      <p>Components are reusable UI pieces.</p>
      <PrimaryButton>Continue</PrimaryButton>
    </Card>
  );
}
```

* `Card` accepts `children` so you can put any nested content inside.
* `PrimaryButton` encapsulates styling so you reuse it consistently.

## Common Pitfalls and Gotchas

* Returning siblings without a wrapper ‚Üí wrap in a fragment `<>...</>`
* Naming components with lowercase ‚Üí must be PascalCase
* Side effects in the render body ‚Üí avoid; rendering should be pure
* Using `class` in JSX ‚Üí use `className`

## Style and File Organization Tips

* One component per file when it grows beyond a few lines
* Name files like `ProfileCard.jsx`, keep related CSS nearby
* Keep components small and focused on a single responsibility
