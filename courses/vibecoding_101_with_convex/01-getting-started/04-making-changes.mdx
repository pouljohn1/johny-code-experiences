## The Power of Iteration

**Iteration** means making small improvements over time. You don't need to get everything perfect on the first try. Instead:

1. Build something basic
2. Try it out
3. See what could be better
4. Request improvements
5. Repeat!

This is how professional developers work, and it's how you'll work with Convex Chef.

## Making Your First Change

Let's make your todo list more useful. We'll add a feature to see when tasks were created.

### Step 1: Decide What You Want

Think about what would make your app better. For example:
- "Show when each task was added"
- "Add a button to clear all completed tasks"
- "Make urgent tasks show in red"

Let's start with the first one.

### Step 2: Ask for the Change

In the prompt bar, type:

> Add a timestamp to each task showing when it was created

Then press Enter or click Generate.

### Step 3: Wait for Chef

Convex Chef will update your app. This usually takes just a few seconds.

### Step 4: Check the Results

Look at your app preview. You should now see timestamps (like "Added 2 minutes ago") next to each task!

Try adding a new task. See the timestamp appear? That's iteration in action! ðŸŽ‰

## Tips for Making Good Changes

### Be Specific

âŒ Vague: "Make it better"  
âœ… Specific: "Add a search box to filter tasks by keyword"

âŒ Vague: "Change the colors"  
âœ… Specific: "Use a blue theme with white text on buttons"

### One Change at a Time

Instead of:
> "Add categories, due dates, priorities, tags, and a calendar view"

Try:
> "Add categories so I can group tasks by project"

Then, after that works:
> "Add due dates to each task"

Smaller changes are easier for Chef to implement correctly.

### Test After Each Change

After Chef makes a change:
1. Look at the preview
2. Click around and test it
3. Make sure it works as expected

If something isn't quite right, just ask for another adjustment!

### Use Examples

If you have a specific vision, describe it:

> "Add a priority field with three options: Low, Medium, High. High priority tasks should have a red border."

The more detail you provide, the closer Chef gets to your vision.

## When Things Don't Work Perfectly

Sometimes Chef might not understand exactly what you want. That's okay! Here's what to do:

### Strategy 1: Clarify Your Request

If the change isn't quite right, explain what's wrong:

> "The timestamps are too small. Make them bigger and easier to read."

### Strategy 2: Try Different Words

If Chef seems confused, rephrase your request:

Instead of: "Make the UI pop"  
Try: "Add colorful buttons and increase the spacing between elements"

### Strategy 3: Break It Down

If a complex change doesn't work, split it into smaller pieces:

Instead of: "Reorganize everything to be more user-friendly"  
Try:
1. "Group related buttons together"
2. "Add clear labels to each section"
3. "Increase the font size for better readability"

### Strategy 4: Ask to Revert

If a change made things worse:

> "Undo the last change and go back to how it was before"
